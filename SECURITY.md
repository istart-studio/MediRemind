## 报告漏洞

我们非常重视安全问题。如果您发现安全漏洞，请发送电子邮件至 [您的邮箱地址]，而不是创建公开的GitHub issue。

请在您的报告中包含以下信息：

1. 漏洞的类型
2. 漏洞的完整路径或位置
3. 任何可能的修复建议
4. 重现步骤或概念验证

我们将尽快回复您的报告，并与您合作解决问题。

## 披露政策

当我们收到安全漏洞报告时，我们将：

1. 确认收到您的报告并开始调查
2. 确定漏洞的范围和严重性
3. 准备修复程序
4. 发布包含修复的新版本
5. 在修复发布后公开披露漏洞（除非有特殊情况）

## 安全最佳实践

作为用户，我们建议您：

1. 始终使用最新版本的应用
2. 不要将应用安装在已越狱或已获得root权限的设备上
3. 定期检查更新
